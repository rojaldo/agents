n8n

Automatización de Workflows

Plataforma de código abierto para automatizar procesos

¿Qué es n8n?

  • Plataforma de automatización de workflows
  • Código abierto y fair-license
  • Interfaz visual drag-and-drop
  • Integración con 400+ aplicaciones
  • Auto-hospedable en servidores propios
  • Sin restricciones de uso comercial

n8n vs Competencia

  • vs Zapier: Menos restricciones, auto-hospedable, más económico
  • vs Make (Integromat): Interfaz más intuitiva, mejor documentación
  • vs Power Automate: Independencia de Microsoft, código abierto
  • vs Custom Code: Más rápido, menos mantenimiento, no requiere devs

Arquitectura y Componentes

  • Backend: Node.js con Express
  • Frontend: Vue.js + UI moderna
  • Base de datos: SQLite (default) o PostgreSQL
  • Ejecución: Queue-based para workflows
  • Webhooks: HTTP listeners para triggers externos

Casos de Uso Comunes

  • Automatización de email marketing
  • Sincronización de datos entre sistemas
  • Procesamiento de formularios web
  • Gestión de redes sociales
  • Automatización de e-commerce
  • Pipelines de datos (ETL)
  • Integración de sistemas legacy

Roadmap de Aprendizaje

  • Fundamentos y UI (2-3 días)
  • Primeros workflows prácticos (3-4 días)
  • Nodos y conectores avanzados (1-2 semanas)
  • Expresiones y transformación de datos (1 semana)
  • Control de flujo y manejo de errores (4-5 días)
  • Integración con APIs externas (1 semana)
  • Deployment y escalabilidad (1 semana)

MÓDULO 1: Fundamentos de n8n

Instalación: Docker (Recomendado)

version: '3.8'
services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=changeme
      - GENERIC_TIMEZONE=Europe/Madrid
    volumes:
      - n8n_data:/home/node/.n8n

Instalación: Docker + PostgreSQL (Producción)

services:
  postgres:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=n8n
      - POSTGRES_PASSWORD=n8n_secure
      - POSTGRES_DB=n8n
    volumes:
      - postgres_data:/var/lib/postgresql/data
  n8n:
    environment:
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=postgres
      - DB_POSTGRESDB_DATABASE=n8n
      - EXECUTIONS_DATA_PRUNE=true

Instalación: Local (Node.js)

  • Requisitos: Node.js 18+ y npm

  • Iniciar: n8n en terminal

  • Ventaja: Fácil para desarrollo local

Conceptos Fundamentales

Workflows

  • Definición: Secuencia de acciones automatizadas
  • Componentes: Nodos conectados entre sí
  • Triggers: Punto de inicio (manual, webhook, schedule)
  • Acciones: Tareas a realizar (enviar email, guardar datos)
  • Flujo: Orden y ramificación de ejecución
  • Estado: Activo (activado) o inactivo (pausado)

Nodos y Conexiones

  • Nodo: Unidad básica que realiza una tarea específica
  • Puertos de entrada: Reciben datos de nodos anteriores
  • Puertos de salida: Envían resultados a nodos siguientes
  • Conexión: Línea entre puertos que transporta datos
  • Data flow: Los datos se propagan de entrada a salida
  • Múltiples conexiones: Un nodo puede conectarse a varios

Triggers (Disparadores)

  • Función: Inician la ejecución del workflow
  • Manual Trigger: Ejecutar manualmente desde UI
  • Webhook: HTTP request externo activa workflow
  • Schedule/Cron: Ejecución programada (cada hora, diaria, etc)
  • Email: Recibir email activa workflow
  • Otros: Cambios en Google Sheets, nuevas filas en DB, etc

Acciones y Datos

  • Acción: Nodo que realiza una tarea específica
  • Ejemplos: Enviar email, guardar en DB, llamar API
  • Datos: Información que fluye entre nodos (JSON)
  • Estructura: Objetos y arrays con campos específicos
  • Transformación: Cambiar formato o valores de datos
  • Validación: Asegurar que datos cumplen formato esperado

Expresiones y Variables

  • Sintaxis: {{$node.nodeName.data.field}}
  • Acceso a datos: $json, $input, $node, $binary
  • Variables de entorno: {{$env.VARIABLE_NAME}}
  • Sistema: $now, $workflow.id, $execution.id
  • Funciones: .toUpperCase(), .trim(), .split()
  • Lógica: Operadores: &&, ||, ?, ternario, etc

Nodos Básicos: Estructura

  • Manual Trigger: Ejecutar manualmente desde botón
  • Set Node: Crear/transformar objeto de datos
  • Function Node: Código JavaScript personalizado
  • HTTP Request: Llamar APIs externas
  • Email Node: Enviar emails
  • Slack Node: Enviar mensajes a Slack

Trigger: Manual Trigger

  • Uso: Ejecutar workflow con click en botón
  • Configuración: Sin parámetros especiales
  • Ideal para: Testing y workflows bajo demanda
  • Inputs: Proporcionar datos JSON de prueba
  • Ejecución: Click en Play/Execute button
  • Salida: Datos que proporciones como input

Trigger: Schedule (Cron)

  • Uso: Ejecutar en horarios específicos
  • Cron Expressions:
    • 0 9 * * * - Todos los días a las 9 AM
    • */15 * * * * - Cada 15 minutos
    • 0 0 1 * * - Primer día de cada mes
  • Timezone: Configurar zona horaria
  • No requiere webhook: Se ejecuta internamente

Trigger: Webhook

  • Uso: HTTP request externo activa workflow
  • URL: https://[tu-n8n]/webhook/[nombre]
  • Método: GET, POST, PUT, DELETE, etc
  • Autenticación: Opcional (API key, token)
  • Datos: Los datos enviados se capturan automáticamente
  • Testing: curl para probar desde terminal

Proyecto Práctico: Notificación por Slack

  • Paso 1: Manual Trigger como entrada
  • Paso 2: Nodo Set para crear mensaje formateado
  • Paso 3: Slack Node para enviar mensaje a canal
  • Paso 4: Conectar todos los nodos secuencialmente
  • Paso 5: Activar y ejecutar para probar
  • Extensión: Agregar condicional IF para validar datos

Buenas Prácticas en Workflows

  • Nombrado: Nombres descriptivos para nodos y workflows
  • Comentarios: Notas en nodos complejos
  • Modularidad: Workflows pequeños y reutilizables
  • Testing: Probar con datos reales antes de activar
  • Manejo de errores: Continue on Error en puntos críticos
  • Organización: Agrupar nodos relacionados visualmente

MÓDULO 4: Expresiones y Variables

Sintaxis de Expresiones

  • Formato: {{expresión}}
  • Acceso simple: {{$node.nombreNodo.data.campo}}
  • Anidado: {{$node.HTTP.data.user.profile.name}}
  • Arrays: {{$node.HTTP.data.items[0].id}}
  • Funciones: {{$node.data.email.toUpperCase()}}
  • Lógica: {{$json.status === 'active' ? 'Sí' : 'No'}}

Variables Especiales

  • $node: Acceder datos de otro nodo
  • $json: Datos del nodo actual
  • $input: Entrada directa del nodo
  • $binary: Datos binarios (archivos)
  • $env: Variables de entorno del sistema
  • $now: Timestamp actual

Más Variables del Sistema

  • $workflow.id: ID del workflow actual
  • $workflow.name: Nombre del workflow
  • $execution.id: ID de esta ejecución
  • $execution.resumeUrl: URL para reanudar ejecución
  • $executionId: Shortcut para execution ID
  • Safe navigation: $node.data?.user?.name

Funciones de String

  • .toUpperCase(): Convertir a mayúsculas
  • .toLowerCase(): Convertir a minúsculas
  • .trim(): Eliminar espacios
  • .split(','): Dividir en array
  • .replace(old, new): Reemplazar texto
  • .substring(0, 5): Extraer parte de string

Funciones de Array

  • .filter(): Filtrar elementos que cumplan condición
  • .map(): Transformar cada elemento
  • .reduce(): Acumular valores
  • .find(): Encontrar primer elemento
  • .join(): Unir elementos en string
  • .includes(): Verificar si contiene elemento

Funciones de Fecha

  • new Date(): Fecha actual
  • .toISOString(): Formato ISO 8601
  • .getTime(): Timestamp en milisegundos
  • .toLocaleDateString(): Fecha formateada local
  • Parsing: new Date(string).toISOString()
  • Comparación: Diferencia entre fechas

Operadores y Lógica

  • Comparación: ===, !==, >, <, >=, <=
  • Lógica: &&, ||, !
  • Ternario: condición ? true : false
  • Nullish: valor ?? default
  • Spread: [...array1, ...array2]
  • Desestructuración: {name, email} = object

Casos de Uso Comunes

  • Validar email: $json.email.includes('@')
  • Convertir tipo: parseInt($json.count)
  • Valor por defecto: $json.name || "Sin nombre"
  • Concatenar: `Hola ${$json.name}`
  • Condicional complejo: Combinar múltiples operadores
  • Formato dinámico: Construir URLs y templates

MÓDULO 5: Control de Flujo

Condicionales: IF Node

  • Uso: Bifurcar workflow según condición
  • Sintaxis:
    • Campo: $json.status
    • Operador: equals, contains, >, <, empty, etc
    • Valor: texto o número
  • Salidas: True branch y False branch
  • Múltiples condiciones: AND y OR

Switch y Múltiples Rutas

  • Uso: Más de 2 opciones (vs IF que es solo sí/no)
  • Estructura:
    • Expression: Campo a evaluar
    • Cases: Valor esperado → ruta
    • Default: Ruta si no coincide ninguno
  • Ejemplo: Status puede ser pending, approved, rejected
  • Ventaja: Más limpio que múltiples IF anidados

Loops e Iteración

  • Loop Node:
    • Procesar array elemento a elemento
    • Acumular resultados
    • Ejecutar nodos internos para cada item
  • Casos de uso:
    • Enviar email a múltiples destinatarios
    • Crear registro en DB para cada fila CSV
    • Procesar items en paralelo

Merge y Combinación de Datos

  • Merge Node:
    • Combinar arrays de múltiples fuentes
    • Modos: Append, Combine, Merge objects
  • Casos:
    • Unir resultados de 2 APIs diferentes
    • Agregar datos de múltiples formatos
    • Crear dataset consolidado

Manejo de Errores: Continue on Fail

  • Opción: "Continue on Fail" en nodo
  • Comportamiento: Si falla, continuar con siguiente nodo
  • Salida en error: Contains error message
  • Casos de uso:
    • API puede no responder
    • Datos opcionales pueden no existir
    • No detener workflow por error no crítico

Reintentos Automáticos

  • Configuración: Retry en cada nodo
  • Parámetros:
    • Max Retries: Número de intentos
    • Wait Between Retries: Milisegundos entre intentos
    • Multiplier: Incremento exponencial
  • Mejores prácticas:
    • APIs externas pueden tener timeouts
    • Conexiones de red pueden ser inestables
    • Usar para errores transitorios

Notificaciones de Error

  • Error Workflow Node:
    • Capturar y procesar errores
    • Enviar notificaciones (email, Slack)
    • Logging personalizado
  • Casos:
    • Notificar a admin si falla crítico
    • Guardar error en DB para auditoría
    • Intentar fallback automático

MÓDULO 6: Gestión de Datos

Lectura de Datos

  • HTTP Request: Llamar APIs GET para obtener datos
  • Database: SELECT queries a PostgreSQL, MySQL, SQLite
  • Google Sheets: Leer filas y columnas específicas
  • CSV/Excel: Importar y parsear archivos
  • Webhooks: Recibir datos en payload
  • Emails: Extraer contenido de attachments

Escritura de Datos

  • INSERT en DB: Agregar nuevos registros
  • HTTP POST: Enviar datos a APIs
  • Google Sheets APPEND: Agregar filas nuevas
  • Archivos: Guardar CSV, JSON, texto
  • Webhooks: Enviar respuesta HTTP
  • Transacciones: Garantizar integridad de datos

UPDATE y DELETE

  • UPDATE en DB:
    • Modificar registros existentes
    • WHERE clause para filtrar qué actualizar
    • Validar que UPDATE afectó registros esperados
  • DELETE en DB:
    • Eliminar registros
    • CUIDADO: Validar condición antes de delete
    • Mejor: Soft delete (marcar como eliminado)

Trabajar con Bases de Datos

  • PostgreSQL: Production-grade, recomendado
  • MySQL: Alternativa popular
  • SQLite: Default, ligero, local
  • Connection Pool: Optimizar múltiples conexiones
  • Credenciales: Guardar en Credentials de n8n
  • Queries: SQL directo o queries parametrizadas

CSV y Excel

  • Lectura: Parse CSV/Excel a JSON array
  • Estructura: Primera fila como headers
  • Mapping: Mapear columnas a campos JSON
  • Escritura: Convertir JSON a CSV/Excel
  • Batch: Procesar miles de filas eficientemente
  • Descarga: Generar archivo para descargar

MÓDULO 7: Webhooks y APIs

Crear Webhooks en n8n

  • Trigger Webhook: HTTP Webhook node en n8n
  • URL generada: https://[tu-n8n]/webhook/[nombre]
  • Método: GET, POST, PUT, DELETE
  • Respuesta: Datos del workflow o mensaje custom
  • Autenticación: Opcional con API Key o token
  • Testing: curl desde terminal o Postman

Consumir APIs Externas

  • HTTP Request Node: El nodo universal para APIs
  • Configuración:
    • Método HTTP (GET, POST, etc)
    • URL del endpoint
    • Headers (Content-Type, Authorization)
    • Query Parameters o Body
  • Respuesta: JSON automáticamente parseado
  • Errores: Capturar HTTP errors

Autenticación en APIs

  • API Key: Token simple en header o parameter
  • Basic Auth: Base64 encoded user:password
  • Bearer Token: JWT o OAuth token
  • OAuth 2.0: Flow seguro (ya integrado en n8n)
  • Custom Headers: Parámetros específicos de API
  • Almacenamiento: Credentials cifrados en n8n

APIs Públicas Populares

  • OpenWeatherMap: Datos meteorológicos
  • Stripe: Procesar pagos
  • GitHub: Gestión de repositorios
  • Google Maps: Geolocalización y mapas
  • NewsAPI: Agregador de noticias
  • Twilio: SMS y llamadas

Rate Limiting y Throttling

  • Rate Limit: Límite de requests por minuto/hora
  • Throttling: Espaciar requests automáticamente
  • Implementación:
    • Detectar headers: X-RateLimit-Remaining
    • Esperar antes de siguiente request
    • Exponential backoff en errores 429
  • Mejores prácticas:
    • Documentación API especifica límites
    • Monitorear uso de quota
    • Alertar si se acerca al límite

MÓDULO 9: Nodos Avanzados

Code Node (JavaScript)

  • Uso: Ejecutar JavaScript personalizado
  • Acceso:
    • items: Array de datos de entrada
    • $env: Variables de entorno
    • $node: Datos de otros nodos
  • Casos:
    • Transformación compleja de datos
    • Cálculos matemáticos
    • Lógica de negocio personalizada

Function Node

  • vs Code Node: Alternativa más simple
  • Lenguaje: JavaScript o TypeScript
  • Retorno: return items; para pasar al siguiente nodo
  • Debugging: console.log() available
  • Casos:
    • Mapeo de campos
    • Filtrado de datos
    • Formato de salida

Python Node (Docker)

  • Requisito: Docker instalado y ejecutándose
  • Caso: Ejecutar scripts Python complejos
  • Limitaciones:
    • Más lento que JavaScript
    • Requiere Docker overhead
  • Ventajas:
    • Librerías Python (pandas, numpy, etc)
    • Científico/análisis avanzado

SQLite Node

  • Uso: Ejecutar queries SQL en SQLite local
  • Ventajas:
    • No requiere servidor DB externo
    • Persistencia automática en n8n
    • Rápido para datos pequeños/medianos
  • Casos:
    • Cache de datos
    • Tablas de referencia
    • Estado del workflow

Nodos de Programación

MÓDULO 12: Integración Avanzada

Webhooks Bidireccionales

  • Concepto: Dos sistemas se notifican mutuamente
  • Caso 1: n8n recibe datos → envía a API externa
  • Caso 2: API externa llama n8n → n8n responde
  • Sincronización: En tiempo real, no polling
  • Desafíos: Manejo de duplicados, orden de eventos
  • Solución: Timestamp, idempotency keys, deduplication

APIs GraphQL

  • vs REST: Más flexible, pedir solo datos necesarios
  • HTTP Request Node: Soporta GraphQL con POST
  • Query: Especificar qué campos necesitas
  • Variables: Pasar datos dinámicos en query
  • Casos: GitHub API, Shopify, etc
  • Ejemplo: GraphQL query en body de HTTP Request

Integraciones Complejas

  • Multi-step: Workflows con 50+ nodos
  • Dependencias: Un paso depende de resultados anteriores
  • Loops anidados: Iteración dentro de iteración
  • Condicionales complejos: Lógica de negocio intrincada
  • Performance: Puede ser lento, optimizar es crítico
  • Mantenibilidad: Documentar bien, refactorizar regularmente

Webhook Templates y Patterns

  • Template: Patrón reutilizable para webhooks
  • Validación: Verificar origen, signature, token
  • Parsing: Extraer datos relevantes del payload
  • Deduplicación: Evitar procesar mismo evento 2 veces
  • Retry Logic: Si falla, reintentar automáticamente
  • Response: 200 OK rápidamente, procesar asincrónico

MÓDULO 13: Instalación y Deployment

Instalación Local

  • Node.js 18+: Requisito previo
  • Instalación: npm install -g n8n
  • Ejecución: n8n en terminal
  • Acceso: http://localhost:5678
  • Ventajas: Rápido, fácil para desarrollo
  • Desventajas: No persistente, datos se pierden

Docker Deployment

  • Simplest: docker run -p 5678:5678 n8nio/n8n
  • Persistent: Usar volumes para guardar datos
  • Production: Docker Compose con PostgreSQL
  • Ventajas: Portable, replicable, fácil escalar
  • Variables de entorno: Configuración desde container
  • Networking: Conectar con otras containers

Deployment en Nube

  • AWS EC2: Instancia Linux + Docker
  • DigitalOcean: Simple y económico
  • Railway.app: Deploy con git push
  • Render.com: Alternativa a Heroku
  • n8n Cloud: Solución oficial managed
  • Kubernetes: Para escalabilidad masiva

Configuración de Base de Datos

  • SQLite (Default): Archivo local, no requiere servidor
  • PostgreSQL (Recommended): Production-grade
  • Environment Variable: DB_TYPE=postgresdb
  • Connection String: postgres://user:pass@host:5432/db
  • Migration: Herramientas para migrar de SQLite a Postgres
  • Backup: Incluir DB en backup strategy

Configuración de Persistencia

  • Volumes en Docker: - n8n_data:/home/node/.n8n
  • Datos guardados: Workflows, credenciales, executions
  • Restore: Copiar volume de backup
  • Encryption: Credenciales cifradas automáticamente
  • Archive: Limpiar executions antiguas por espacio
  • S3 Backup: Sincronizar backups a cloud

Escalabilidad Horizontal

Performance Optimization

  • Connection Pooling: Reutilizar conexiones DB
  • Indexing: Índices en DB para queries rápidas
  • Lazy Loading: Cargar datos solo cuando necesario

Monitoring en Producción
